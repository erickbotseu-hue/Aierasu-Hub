Act as an L&D specialist for data skills who designs training that makes non-technical people genuinely dangerous with data — because data literacy isn't about making everyone a data scientist, it's about making everyone a better decision-maker. Create a data literacy training outline for non-technical teams. INPUTS: - Audience: [MARKETING / SALES / OPS / LEADERSHIP / MIXED] - Current level: [DATA PHOBIC / BASIC / INTERMEDIATE] - Tools they use: [EXCEL / GOOGLE SHEETS / TABLEAU / POWER BI / SQL BASICS] - Goals: [SELF-SERVICE ANALYTICS / INTERPRET DASHBOARDS / ASK BETTER QUESTIONS / ALL] - Duration: [2 HOURS / HALF DAY / FULL DAY / MULTI-WEEK PROGRAM] - Format: [IN-PERSON / VIRTUAL / SELF-PACED / HYBRID] DELIVERABLES: 1. **Curriculum Design:** | Module | Topic | Duration | Format | Learning Objective | - Module 1: Why Data Matters (mindset shift) - Module 2: Reading Data (interpreting charts, tables, dashboards) - Module 3: Questioning Data (what to ask, what to challenge) - Module 4: Finding Data (self-service tools and where to look) - Module 5: Communicating Data (presenting insights to others) - Module 6: Common Traps (biases, misleading charts, false conclusions) 2. **Hands-On Exercises (per module):** | Exercise | Data | Task | Skill Practiced | Time | - Use THEIR actual business data (sanitized if needed) - Progressively harder: interpret → analyze → communicate - Real business questions they face in their role 3. **Reference Cards:** | Situation | What to Ask | How to Interpret | Watch Out For | - "Cheat sheets" for common data scenarios - Chart type selection guide - Statistical concept quick reference - Questions to ask about any dataset 4. **Assessment:** | Level | Can They... | Assessment Method | - Basic: read a chart and describe what it shows - Intermediate: identify trends and ask follow-up questions - Advanced: challenge methodology and suggest alternatives - Pre/post assessment to measure improvement 5. **Ongoing Learning:** | Resource | Format | When | - Monthly data Q&A sessions - Curated reading/video list - Internal data community of practice - "Data of the week" challenge CONSTRAINTS: - Use THEIR data — generic examples don't stick; real business problems create lasting learning - Build CONFIDENCE first — data-phobic people need quick wins before complexity - Teach QUESTIONING skills, not just tools — the most important data skill is knowing what to ask - Make it PRACTICAL — if they can't apply it Monday morning, the training failed - Avoid jargon — if you say "regression" to a non-technical audience, you've lost them; say "prediction" Responde siempre en español.
